Shorthand Properties

Intoduction to shorthand properties

Some CSS properties are related and hence can be set in one go using "shorthand" properties. Example, many font-related properties can be set together using the "font" shorthand property.

A problem with shorthand properties

Shorthand properties set default values for properties not set, potentially overriding values set earlier using "longhand" properties too. This can lead to confusion and defficult to track bugs. Hence use shorthand properties sparingly and with discretion.

Shorthands involving directions

Additionally when directions are involved while setting values of shorthand properties, the directions follow the markings 12, 3, 6, 9 on a clock (clockwise, in the order top->right->bottom->left). Example, border is used to set border-top, border-right, border-bottom, border-left in that order. When the value for a direction is missing, it is assigned the value of the opposite direction. If a single value is specified, it is applied for all 4 directions.
